Onboarding Guide

                    
Step 1: Create Project

                    
Use the central selector to add a new project. Fill in GA4/Meta IDs and domains.

                    
                    
Step 2: Generate Version

                    
Saving a config creates a version snapshot. This enables rollbacks and dynamic script delivery.


                    
Step 3: Embed Scripts

                    
Copy the snippets to your funnel pages. Header goes on every page, Thank-you only on the final success page.

                

            

            
            

                

                    
Signal-Path Orchestrator Manual (V3.2)

                    

                        
                        

                            
🚀 Önemli: Bu Sistem Dinamiktir

                            
Her yeni proje için GitHub'a kod pushlamanıza gerek yoktur. Proje ayarlarını burada yapıp Save dedikten sonra Promote butonuna basmanız, ayarları Yedek Sunucuya (Vercel) anında aktarır.

                        


                        
1. Ana Sunucu: Cloudflare (Primary)

                        
Proje Adı: nilufer-script-tracking

                        
    
                            
  • ADMIN_TOKEN ekleme: Settings > Functions > Variables > ADMIN_TOKEN = [Şifreniz].
    • 
                            
  • KV Bağlama: Settings > Functions > KV Namespace Bindings > SIGNAL_KV.
    • 
                        


                        
2. Yedek Sunucu: Vercel (Backup)

                        
Proje Adı: signaltracker-backup

                        
    
                            
  • Kaynak: GitHub deponuzu Vercel'e bağlayın (Ana depoyla aynı kodları içerir).
    • 
                            
  • ADMIN_TOKEN ekleme: Settings > Environment Variables > ADMIN_TOKEN = [Cloudflare ile aynı şifre].
    • 
                            
  • Webhook URL: Configuration sekmesindeki "Backup Webhook" alanına şunu yazın: https://signals.niluferormanlistudio.com/api/sync
    • 
                        


                        
3. İlk Senkronizasyon (Promote)

                        
    
                            
  1. Primary panelde projenizi kaydedin (Save & Deploy).
    2. 
                            
  2. Dashboard'a gidip 🚀 Promote to Backup butonuna basın.
    3. 
                            
  3. "Tracking Pulse" kısmında Backup sütununun yeşil olduğunu doğrulayın.
    4. 
                        


                        

                            Pro Tip: GitHub deponuzun (backup reposu) her zaman ana depomuzdaki main branchi ile güncel (Sync) olduğundan emin olun.
